# Carpeta `entregable/`

Aquí va lo que vas a **subir al aula virtual del Curso 2.4** para que tu trabajo sea evaluado.

## Estructura del entregable

```
entregable/
├── README.md                  # Este archivo
├── system-prompt-final.md     # El system prompt o configuración final de tu bot
├── reflexion.md               # Tu reflexión escrita (200-400 palabras)
└── capturas/                  # 2-3 capturas o transcripciones de tus mejores pruebas
    └── [tus archivos aquí]
```

## Qué evalúa la rúbrica

Cuatro criterios, 25 % cada uno, total 100 %. Aprobación con **nota ≥ 4,0** sobre 7,0.

| Criterio | Qué se evalúa | Dónde se evidencia |
|---|---|---|
| **C1. Pertinencia del caso** | El caso elegido conecta con tu rol institucional real en UMCE y es accionable | `mi-perfil.md` + introducción de `reflexion.md` |
| **C2. Solidez técnica** | El system prompt tiene los 4 componentes (rol/estilo/criterios/restricciones), suficiente densidad (≥300 palabras), restricciones éticas claras | `system-prompt-final.md` |
| **C3. Calidad de las pruebas** | Las conversaciones de prueba demuestran que el bot funciona en caso típico y caso límite | `capturas/` |
| **C4. Profundidad de la reflexión** | Tu reflexión articula límites éticos, aplicabilidad y ajustes hechos durante la construcción | `reflexion.md` |

> Detalle completo de cada criterio en `docs/rubrica-resumen.md`.

## Checklist antes de entregar

### Tu system prompt final (`system-prompt-final.md`)
- [ ] Tiene los 4 componentes claramente etiquetados (ROL / ESTILO / CRITERIOS / RESTRICCIONES).
- [ ] Suma al menos 300 palabras.
- [ ] Restricciones incluyen al menos 3 reglas duras (sobre datos sensibles, derivación a humanos, no inventar).
- [ ] No tiene datos sensibles ni nombres reales de estudiantes/colegas.

### Tus pruebas (`capturas/`)
- [ ] Al menos 2 conversaciones: una típica y una de borde (fuera de rol).
- [ ] Datos personales anonimizados en todas las capturas.
- [ ] Cada captura está acompañada por una breve nota tuya sobre qué muestra y qué aprendiste.

### Tu reflexión (`reflexion.md`)
- [ ] Tiene entre 200 y 400 palabras.
- [ ] Articula: caso elegido + por qué te sirve + límites éticos detectados + qué ajustes hiciste.
- [ ] Identifica al menos un límite ético concreto del uso de IA en tu rol.
- [ ] Termina con una proyección honesta: ¿lo vas a usar realmente? ¿En qué condiciones?

## Cómo se sube al aula virtual

1. Cuando termines, comprime esta carpeta `entregable/` en un ZIP.
2. Entra al aula virtual del Curso 2.4 en https://evirtual.umce.cl (link específico en tu correo institucional).
3. Busca la actividad "Entrega final del curso".
4. Sube el ZIP antes del plazo de cierre de tu grupo:
   - **Grupo 1**: domingo 7 de junio de 2026, 23:59 hrs.
   - **Grupo 2**: domingo 2 de agosto de 2026, 23:59 hrs.

## Si algo se te quedó pendiente

Mejor entregar algo mínimo viable que no entregar nada. Si solo tienes el system prompt y una prueba, súbelo igual — vas a obtener parte del puntaje y eso es mejor que ausentarse.

Si necesitas extensión por algún motivo, escríbele al equipo UDFV (`udfv@umce.cl`) **antes** del plazo, no después.
